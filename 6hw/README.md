# Итоговый проект 
## Обзор выбранных данных
Для выполнения работы был выбран датасет(временной ряд) "Time Series starter dataset" https://www.kaggle.com/datasets/podsyp/time-series-starter-dataset/data

## Структура данных

Датасет содержит помесячные данные о деятельности компании за период с января 2015 по январь 2022 года. Ниже представлено описание колонок:

| Название колонки | Тип данных | Описание | Пример значения | Примечание |
|------------------|------------|----------|-----------------|------------|
| `Period` | Дата (datetime) | Временная метка записи (день.месяц.год) | 01.01.2015 | Используется как индекс временного ряда |
| `Revenue` | Числовой (float) | Выручка компании за месяц (целевая переменная) | 14 000 000 | Содержит 32 пропущенных значения (требовалась интерполяция) |
| `Sales` | Числовой (float) | Объем продаж за месяц | 58 800 000 | Содержит 32 пропущенных значения |
| `avg_cost` | Числовой (float) | Средняя себестоимость продукции | 8 314 | Содержит 32 пропущенных значения |
| `avg_employees` | Числовой (float) | Среднее количество сотрудников в регионе за год | 38 100 | Содержит 32 пропущенных значения |

## Обработка пропущенных значений

В ходе первичного анализа данных было выявлено значительное количество пропусков:

| Колонка | Количество пропусков | Процент от общего объема |
|---------|---------------------|--------------------------|
| Revenue | 32 | ~38% |
| Sales | 32 | ~38% |
| avg_cost | 32 | ~38% |
| avg_employees | 32 | ~38% |

Примечание: пропуски были обработаны методом линейной интерполяции для сохранения временной структуры данных.

## Построение прогнозных моделей

Было реализовано и ghjdtltyj сравненbt 6 моделей для прогнозирования:
1. SARIMA
2. ARIMA
3. Holt-Winters (ETS)
4. Linear Regression
5. Random Forest
6. XGBoost

**Подход к созданию признаков для ML-моделей:**
- Созданы лаговые переменные (lag_1 до lag_12)
- Добавлены временные признаки: месяц, квартал, год
- Использовано скользящее среднее для сглаживания
